# How to Import CSV Directly into Supabase

## 📋 The Challenge

Your current CSV structure:
```
STD - 10 ICSE
,PHYSICS,TASK 1,TASK 2,...,TASK 13
1,FORCE,Textbook Reading,Textbook Examples,...
2,WORK ENERGY,Textbook Reading,...
```

But Supabase tables are **normalized** (separate tables):
- `subjects` - stores subjects
- `topics` - stores topics/chapters
- `tasks` - stores individual tasks

So you need to **split the CSV into 3 separate files**.

---

## 🔄 Step 1: Create Subjects CSV

**File**: `subjects_for_import.csv`

```csv
id,name,batch,description,is_active,created_at,updated_at
,PHYSICS,ICSE 10,Imported from curriculum CSV,true,NOW(),NOW()
,CHEMISTRY,ICSE 10,Imported from curriculum CSV,true,NOW(),NOW()
,BIOLOGY,ICSE 10,Imported from curriculum CSV,true,NOW(),NOW()
,ENGLISH,ICSE 10,Imported from curriculum CSV,true,NOW(),NOW()
,HISTORY,ICSE 10,Imported from curriculum CSV,true,NOW(),NOW()
,GEOGRAPHY,ICSE 10,Imported from curriculum CSV,true,NOW(),NOW()
,CIVICS,ICSE 10,Imported from curriculum CSV,true,NOW(),NOW()
```

**How to create**:
1. Open your original CSV in Excel/Google Sheets
2. Look at each subject name (PHYSICS, CHEMISTRY, etc.)
3. Create a new sheet with columns: `id,name,batch,description,is_active,created_at,updated_at`
4. Leave `id` blank (Supabase will auto-generate)
5. Fill in subject names and batch

## 🔄 Step 2: Create Topics CSV

**File**: `topics_for_import.csv`

```csv
id,subject_id,title,order_index,description,created_at,updated_at
,<PHYSICS_ID>,FORCE,1,Imported from CSV,NOW(),NOW()
,<PHYSICS_ID>,WORK ENERGY POWER,2,Imported from CSV,NOW(),NOW()
,<PHYSICS_ID>,MACHINE,3,Imported from CSV,NOW(),NOW()
,<CHEMISTRY_ID>,PERIODIC TABLE,1,Imported from CSV,NOW(),NOW()
,<CHEMISTRY_ID>,CHEMICAL BONDING,2,Imported from CSV,NOW(),NOW()
```

**How to create**:
1. After importing subjects, note down their IDs from Supabase
2. For each topic in your original CSV:
   - Get the subject name
   - Look up that subject's ID
   - Create a row with: `id, <subject_id>, topic_name, order_number`

**How to create**:
1. After importing topics, note down their IDs

   - Find which topic it belongs to
   - Get that topic's ID
   - Create a row with task title and order (1-13)

## 📲 How to Import into Supabase

### **Step 1: Go to Supabase Dashboard**
1. Open: https://app.supabase.com
2. Select your project
3. Go to **SQL Editor** or **Table Editor**

### **Step 2: Import Subjects First**
1. Go to **subjects** table
2. Click **"..."** menu → **Import data** → **CSV**
3. Upload `subjects_for_import.csv`
4. Map columns correctly
5. Click **Import**
6. ✅ Note down the subject IDs

### **Step 3: Import Topics**
1. Go to **topics** table
2. Click **"..."** menu → **Import data** → **CSV**
3. Upload `topics_for_import.csv`
4. **Replace `<SUBJECT_ID>` with actual IDs** from step 2
5. Map columns correctly
6. Click **Import**
7. ✅ Note down the topic IDs

### **Step 4: Import Tasks**
1. Go to **tasks** table
2. Click **"..."** menu → **Import data** → **CSV**
3. Upload `tasks_for_import.csv`
4. **Replace `<TOPIC_ID>` with actual IDs** from step 3
5. Map columns correctly
6. Click **Import**

### **Step 5: Create Student Progress**
Use SQL in Supabase SQL Editor:
```sql
INSERT INTO student_progress (id, student_id, task_id, is_completed, created_at, updated_at)
SELECT 
    gen_random_uuid(),
    s.id,
    t.id,
    false,
    NOW(),
    NOW()
FROM students s
JOIN tasks t ON true
JOIN topics tp ON t.topic_id = tp.id
JOIN subjects subj ON tp.subject_id = subj.id
WHERE s.batch = subj.batch
AND NOT EXISTS (
    SELECT 1 FROM student_progress 
    WHERE student_id = s.id AND task_id = t.id
);
```

## ⚙️ CSV Format Rules for Supabase

✅ **Do**:
- Use correct column names (match Supabase table columns exactly)
- Leave `id` empty (auto-generated by Supabase)
- Use `true`/`false` for booleans
- Use `NOW()` for timestamps
- Use comma separator

❌ **Don't**:
- Include extra columns not in the table
- Leave required columns empty (except `id`)
- Use quotes around UUID values
- Use `1`/`0` for booleans (use `true`/`false`)

---

## 🔑 Important: UUID References

When you see `<SUBJECT_ID>` or `<TOPIC_ID>`, replace with actual UUIDs like:
```
,4f5a3c2e-1b8f-4d6a-9e2c-7a1b5c3d8f2a,FORCE,1,Imported,NOW(),NOW()
```

Not with placeholder text!

3. **Import Tasks**: `tasks_for_import.csv`
   - Replace `<TOPIC_ID_1>` with `uuid-789` (Force)
   - Replace `<TOPIC_ID_2>` with `uuid-999` (Periodic Table)
   - Import all tasks

4. **Run SQL**: Create all student progress records

## 🆘 Troubleshooting

**Error: "Column not found"**
- Check column names match Supabase table exactly
- Example: use `created_at`, not `CreatedAt`

**Error: "Foreign key constraint failed"**
- Make sure you imported parent table first
- Make sure UUIDs are correctly referenced

**Error: "Duplicate entry"**
- Check if data already exists
- Use Supabase's "upsert" option if available
